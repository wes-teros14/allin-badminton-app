## v1 Requirements

### Finance Allocation Mode

- [x] **FIN-01**: QM can keep using the current automatic shuttle allocation behavior unchanged when auto allocation is enabled. - Completed in Phase 16
- [x] **FIN-02**: QM can switch shuttle usage entry between automatic allocation and manual allocation from the session finance page. - Completed in Phase 16
- [x] **FIN-03**: QM can save shuttle usage in auto mode by entering total shuttles used and letting the existing allocation rules decide the batch breakdown. - Completed in Phase 16

### Manual Batch Selection

- [x] **MAN-01**: QM can search available shuttle batches by brand when adding a manual allocation row. - Completed in Phase 17
- [x] **MAN-02**: QM can select multiple shuttle batches for one session's manual shuttle allocation. - Completed in Phase 17
- [x] **MAN-03**: QM can see inventory-style batch details for each available or selected batch, including tube ID, brand, shuttles remaining, cost per tube, and notes when present. - Completed in Phase 17
- [x] **MAN-04**: QM can enter an exact shuttle count for each selected batch in manual mode. - Completed in Phase 17
- [x] **MAN-05**: Session total shuttle usage is auto-calculated from the selected batch counts in manual mode, with no separate total shuttle input. - Completed in Phase 17
- [x] **MAN-06**: QM can reopen a session finance record with saved manual allocation rows and edit those rows before saving again. - Completed in Phase 17

### Validation And Compatibility

- [x] **VAL-01**: QM can save manual shuttle usage only when at least one batch is selected. - Verified in Phase 18
- [x] **VAL-02**: QM can save manual shuttle usage only when no batch is selected more than once. - Verified in Phase 18
- [x] **VAL-03**: QM can save manual shuttle usage only when every per-batch shuttle count is a whole number greater than 0 and does not exceed that batch's available stock at save time. - Verified in Phase 18
- [x] **COMP-01**: QM can view correct shuttle cost, total logged shuttles, and batch allocation details after saving either automatic or manual shuttle usage. - Verified in Phase 18
- [x] **COMP-02**: Existing sessions that rely on the current automatic allocation flow continue to work without any rule or data migration requirement. - Completed in Phase 16

## Out of Scope At Close

| Feature | Reason |
|---------|--------|
| Changing the current automatic allocation rules | Auto mode had to preserve existing behavior exactly |
| Finance completion gates based on a required shuttle target | QM records actual usage rather than satisfying a precomputed total |
| Search filters beyond brand for the manual picker | Brand search was the milestone scope |
| Creating or editing inventory batches from the finance page | Inventory remains the source of truth for batch lifecycle |
