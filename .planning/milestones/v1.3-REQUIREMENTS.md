# Requirements Archive: v1.3 Split Match Scoring

**Archived:** 2026-05-23
**Milestone Status:** SHIPPED

This file preserves the final v1.3 requirement set at milestone close.

## Final Requirement Outcomes

### Match Format

- [x] **FMT-01**: Admin can enable or disable split-match scoring for a session with a single session-level checkbox/toggle.
- [x] **FMT-02**: Sessions with split-match scoring disabled keep the current one-result match finish behavior.
- [x] **FMT-03**: Sessions with split-match scoring enabled require two game winners when finishing a scheduled match.

### Result Recording

- [x] **RES-01**: Admin/live board can record a `2-0` split result for either team.
- [x] **RES-02**: Admin/live board can record a `1-1` split result as a valid final result.
- [x] **RES-03**: Split results persist as two game-level result records for the same scheduled match.
- [x] **RES-04**: Duplicate split-game result records are prevented for the same match and game number.

### Stats And Leaderboards

- [x] **STAT-01**: A `2-0` result counts as two wins for each player on the winning team.
- [x] **STAT-02**: A `1-1` result counts as one win for each team's players.
- [x] **STAT-03**: Session leaderboards, today leaderboard, player schedule, player profile stats, and all-time leaderboard aggregate all game-level result records correctly.

### Compatibility

- [x] **COMP-01**: Existing completed matches without split-game data continue to display and count as one-game results.
- [x] **COMP-02**: Existing session lifecycle, queue advancement, court assignment, and realtime updates keep working in both scoring modes.
