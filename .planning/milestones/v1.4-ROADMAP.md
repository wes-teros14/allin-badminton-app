# Milestone v1.4: Finance Manual Shuttle Allocation

**Status:** SHIPPED 2026-05-25
**Phases:** 16-18
**Total Plans:** 9

## Overview

Delivered manual shuttle allocation on top of the existing session finance flow without changing the current automatic allocation behavior. The milestone added allocation-mode persistence, a searchable batch picker, an editable manual allocation table, save-time validation, and browser-backed regression coverage for both auto and manual finance paths.

## Phases

### Phase 16: Finance Allocation Mode Foundation

**Goal**: Session finance supports explicit auto/manual allocation modes without changing the current automatic shuttle allocation behavior  
**Depends on**: Phase 15  
**Plans**: 3 plans

Plans:

- [x] 16-01: Allocation mode schema
- [x] 16-02: Mode-aware finance hook
- [x] 16-03: Finance mode UI

**Details:**
1. Auto mode continues to use the current allocation rules and save path without changing saved results or cost calculations.
2. Finance data structures and save handlers can represent both auto allocation input and manual per-batch allocation input.
3. Existing finance sessions that use the automatic flow remain editable and readable without migration work.

### Phase 17: Manual Batch Allocation UI

**Goal**: QM can manually build a shuttle allocation from searchable inventory batches with inventory-style batch details  
**Depends on**: Phase 16  
**Plans**: 3 plans

Plans:

- [x] 17-01: Manual finance hook enrichment
- [x] 17-02: Manual batch picker dialog
- [x] 17-03: Manual allocation editor

**Details:**
1. QM can switch to manual mode and search available batches by brand from the finance page.
2. QM can select multiple batches and see the same identifying details already used in inventory.
3. QM can enter per-batch shuttle counts, see the total auto-calculate, and reload an existing manual allocation for editing.

### Phase 18: Validation And Finance Regression Coverage

**Goal**: Manual allocation is stock-safe, duplicate-safe, and verified without regressing current finance behavior  
**Depends on**: Phase 17  
**Plans**: 3 plans

Plans:

- [x] 18-01: Manual validation hook hardening
- [x] 18-02: Inline finance validation UI
- [x] 18-03: Browser finance allocation regression coverage

**Details:**
1. Manual mode blocks invalid saves for empty allocation, duplicate batch selection, non-positive counts, and counts above available stock.
2. Saved allocations in both auto and manual modes produce correct total shuttles, shuttle cost, and batch allocation display.
3. Unit and browser coverage exist for mode switching, brand search, manual row editing, and automatic allocation regression paths.

## Milestone Summary

**Key Decisions:**

- Manual finance allocation remains optional and preserves the existing automatic allocation path by default.
- Allocation mode is stored on the session finance record and normalized to `auto` for compatibility.
- Manual batch selection reuses searchable picker patterns and inventory-style batch details already familiar in the app.
- Manual allocation validation is enforced at the save seam as well as in the UI so invalid payloads never reach persistence.

**Issues Resolved:**

- Added a durable contract for switching between automatic and manual shuttle allocation.
- Replaced the manual-mode placeholder with a real picker and editable per-batch allocation flow.
- Closed the finance regression gap with unit coverage plus real Supabase-backed Playwright coverage.
- Reconciled stale closeout state by verifying Phase 18 through UAT and updating milestone planning artifacts.

**Issues Deferred:**

- FIN-F01: Session-to-session profit trend
- FIN-F02: Low stock alert
- FIN-F03: Shuttle sell-price tracking
- INV-F01: Batch quality notes
- INV-F02: Batch expiry tracking

**Technical Debt Incurred:**

- Supabase migration execution on Windows still depends on the Dashboard SQL Editor instead of the CLI.
- Milestone closeout still required manual archival because `gsd-sdk` was not available in this shell.
