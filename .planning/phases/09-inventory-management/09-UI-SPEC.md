- Touch targets (Add Batch button, dialog close): minimum `h-8` (32px) per existing button component defaults. Mobile tap target meets 44px via vertical page padding context.

- Max content width: `max-w-lg mx-auto` — matches AdminView layout constraint.

## Color

All values reference CSS custom properties from `index.css`. Dark mode values apply automatically via `.dark` class.

| Role | CSS Variable | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `--background` | `#0F0A18` | Page background, InventoryView wrapper |
| Secondary (30%) | `--card` | `#1A1025` | Add Batch Card wrapper, table container |
| Accent (10%) | `--primary` | `#6F3E87` | "Add Batch" CTA button background only |
| Muted surface | `--muted` | `#1E1230` | Depleted row background tint, empty state area |
| Muted text | `--muted-foreground` | `#B39DBB` | Depleted badge text, empty state copy, notes column text |
| Border | `--border` | `#2D1A40` | Table row dividers, card ring |
| Destructive | `--destructive` | `#DC595E` | Not used in Phase 9 (no deletion, no destructive action) |

### Accent Reserved For

Accent (`--primary`, `#6F3E87`) is reserved exclusively for:
1. The "Add Batch" primary CTA button (default variant)
2. Focus ring on form inputs (`--ring: #6F3E87` in dark mode)

Accent is NOT used for: table rows, badge fills, headings, links, or hover states.

### Depleted Badge

Use shadcn `Badge` with `variant="secondary"`.

- Background: `bg-muted` (`#1E1230` in dark)
- Text: `text-muted-foreground` (`#B39DBB` in dark)
- This produces a visually recessed, gray-purple muted badge that signals "inactive" without using destructive red.
- Rationale: `variant="secondary"` maps to `--secondary` / `--secondary-foreground` which in dark mode reads as muted purple-gray — correct visual weight for "still visible but depleted."

### Active Batch Indicator

No special color. Active batches (stock > 0) render with default `--foreground` text on `--card` background — no badge needed.

### Table Column Order

Exact column order (from D-03):

| # | Column Header | Data | Alignment |
|---|--------------|------|-----------|
| 1 | Tube ID | `T-1001` format | Left |
| 2 | Brand | Brand text | Left |
| 3 | Tubes Bought | Integer | Right |
| 4 | Stock Remaining | Integer (computed) | Right |
| 5 | Cost/Tube | `₱85.00` (Intl.NumberFormat) | Right |
| 6 | Notes | Optional text, truncated | Left |

Depleted rows (Stock Remaining = 0): render a `Badge variant="secondary"` with text "Depleted" in the Stock Remaining cell, replacing the "0" value. Row background: `bg-muted/30` to visually recede without hiding.

Sorted: `ORDER BY cost_per_tube ASC` (cheapest first) — applied in the `useShuttleBatches` hook, not client-side.

### Add Batch Dialog

- Trigger: "Add Batch" button in page header
- Width: default Dialog width (matches shadcn default — `sm:max-w-[425px]`)
- Fields stacked vertically with `gap-4`
- Field order: Brand → Tubes Bought → Cost per Tube → Notes
- Notes field: `<textarea>` variant or plain `<Input>` (single line acceptable for notes)
- Submit button: full width inside `DialogFooter`, default variant, `size="default"`
- Validation: React Hook Form + Zod, inline error messages below each field using `text-xs text-destructive` (or `aria-invalid` on inputs — matches existing button `aria-invalid` pattern)
- On success: close dialog, reset form, fire success toast, refresh batch list

## Accessibility Contract

- All form inputs have associated `<Label>` components linked via `htmlFor` / `id`.
- Dialog uses `DialogTitle` and `DialogDescription` for screen reader announcement.
- Empty state `Package` icon is `aria-hidden="true"` — decorative only.
- Depleted badge `aria-label="Depleted batch"` for screen readers that may not read badge text in table context.
- Table uses semantic `<table>`, `<thead>`, `<tbody>` via shadcn Table primitives.
- "Add Batch" button has `aria-label="Add shuttle batch"` if icon-only variant is ever used (full label version preferred).
