## Spacing Scale

Identical to Phase 9 — all multiples of 4, expressed as Tailwind utility classes.

| Token | Value | Tailwind Class | Usage |
|-------|-------|---------------|-------|
| xs | 4px | `gap-1` / `p-1` | Icon gaps, inline label spacing |
| sm | 8px | `gap-2` / `p-2` | Compact element spacing, input prefix |
| md | 16px | `gap-4` / `p-4` | Default card padding, form field spacing |
| lg | 24px | `gap-6` / `p-6` | Page section padding (matches AdminView `p-6`) |
| xl | 32px | `gap-8` / `p-8` | Layout outer padding (not expected in this phase) |
| 2xl | 48px | `gap-12` | Major section breaks (not expected) |
| 3xl | 64px | `gap-16` | Page-level spacing (not expected) |

Exceptions:
- P&L summary card profit/loss line: `text-lg font-semibold` for visual emphasis over standard body size — typography exception, not a spacing exception.
- Touch targets: minimum `h-8` (32px) per existing button defaults.
- Max content width: `max-w-lg mx-auto` — matches AdminView and InventoryView layout constraint.
- Section dividers between the three Finance Detail sections: `space-y-6` between Shuttle Usage, Court Cost, and P&L Summary cards.

All values reference CSS custom properties from `index.css`. Dark mode values apply automatically via `.dark` class. Finance-specific semantic additions are noted.

| Role | CSS Variable | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `--background` | `#0F0A18` | Page background for both Finance views |
| Secondary (30%) | `--card` | `#1A1025` | Finance list card, section cards on detail view |
| Accent (10%) | `--primary` | `#6F3E87` | Save buttons (Shuttle Usage save, Court Cost save) |
| Muted surface | `--muted` | `#1E1230` | Loading skeleton, empty state area, shuttle usage breakdown table background |
| Muted text | `--muted-foreground` | `#B39DBB` | Empty state copy, secondary P&L labels (Revenue, Shuttle Cost, Court Cost) |
| Border | `--border` | `#2D1A40` | Table row dividers, card ring |
| Destructive | `--destructive` | `#DC595E` | Net Loss value (profit < 0), re-log warning text |

### Finance-Specific Semantic Colors (D-11)

Two new semantic usages for this phase — not present in Phase 9:

| Semantic Role | Tailwind Class | CSS Basis | Applied To |
|---------------|---------------|-----------|-----------|
| Profit (≥ 0) | `text-green-500` | Tailwind green-500 | Net Profit value in P&L card; P&L column cells in Finance List where value ≥ 0 |
| Loss (< 0) | `text-destructive` | `--destructive` (#DC595E) | Net Loss value in P&L card; P&L column cells in Finance List where value < 0 |

Rationale: `text-green-500` is Tailwind's semantic green (does not conflict with shadcn CSS variables). `text-destructive` reuses the existing destructive token — consistent with shadcn patterns.

### Accent Reserved For

Accent (`--primary`, `#6F3E87`) is reserved exclusively for:
1. "Save" button on Shuttle Usage section (primary CTA per section)
2. "Save" button on Court Cost section (primary CTA per section)
3. Focus ring on form inputs (`--ring: #6F3E87` in dark mode)

Accent is NOT used for: table rows, P&L values, page titles (page titles use `text-primary` which is the same token — this is acceptable as it is the existing heading pattern), or navigation active states.

### Destructive Actions

Re-logging shuttle usage (overwriting existing data) is the closest to destructive in this phase. No deletion of sessions.

| Action | Approach |
|--------|----------|
| Re-logging shuttle usage when data already exists | No separate confirmation dialog. The "Update Usage" button label signals intent. A `toast.success("Shuttle usage updated.")` confirms completion. The previous allocation is silently replaced (delete-then-insert). No modal required — admin explicitly typed a new number and pressed Save. |

No other destructive actions in Phase 10.

## Accessibility Contract

- All form inputs have associated `<Label>` components linked via `htmlFor` / `id`.
- Back navigation link has discernible text ("Back to Finance") — not icon-only.
- Empty state icon (`ReceiptText`) is `aria-hidden="true"` — decorative only.
- Profit/loss color coding is supplemented by the label change ("Net Profit" vs "Net Loss") — not color-only.
- Finance List table uses semantic `<table>`, `<thead>`, `<tbody>` via shadcn Table primitives.
- Clickable TableRows use `onClick` with keyboard accessibility — add `role="button"` or use `<Link>` wrapper if full keyboard nav is needed.
- P&L values use `aria-label` format: `aria-label="Net Profit: ₱1,200.00"` or `aria-label="Net Loss: ₱300.00"` for screen reader context on color-coded spans.

## `formatPeso` Reuse

Do not duplicate `formatPeso`. Extract to a shared utility or import directly from `InventoryView.tsx`:

```ts
// badminton-v2/src/utils/formatPeso.ts (new shared util)
export const formatPeso = (value: number) =>
  new Intl.NumberFormat('en-PH', {
    style: 'currency',
    currency: 'PHP',
    minimumFractionDigits: 2,
  }).format(value)
```

Both `InventoryView.tsx` and the new Finance views import from this shared util. The executor should extract on first use if not already extracted.
