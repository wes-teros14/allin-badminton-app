==================================================
Source: badminton_app/badminton_app.py

This table answers: "Which variables are active, greyed out, or irrelevant in each mode?"
Note: Greyed-out (disabled) Streamlit widgets still return their default value in code.
      They are blocked from user input only — not from being read.

=== SCORING WEIGHT EFFECTIVENESS ===
Even when Iterative is ON, not all weights actively differentiate trials.
Engine guarantees eliminate some violations before scoring runs.

Weight                  Iterative ON — Fires?   Why
──────────────────────  ──────────────────────  ──────────────────────────────────────────────
p_streak_weight         Rarely (near 0%)        Engine hard-blocks streak violations
p_fairness_weight       Rarely (near 0%)        balance_participation() guarantees gap=0
p_spread_penalty        Never  (0%)             Hard skill gap filter blocks all violations
p_imbalance             YES — active            Level diff between T1/T2 varies per trial
p_repeat_partner        YES — active            Repeat pairs vary per trial (~4 avg)
r_wishlist              YES — active            Wishlist fulfillment varies per trial
