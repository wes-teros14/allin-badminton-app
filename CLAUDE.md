# Badminton V2 ‚Äî Project Instructions

Inherits all rules from global `~/.claude/CLAUDE.md`.

## Project Context

This project uses the **BMAD framework (v6.2.0)** ‚Äî an AI-driven project orchestration system. It provides structured workflows for taking a project idea through analysis, planning, solutioning, and implementation phases using role-based agents.

## Task Management

- Plans go in `tasks/todo.md`
- Lessons learned go in `tasks/lessons.md`

## Bug & Resolution Log

After every bug fixed or issue resolved, add an entry to `tasks/lessons.md` with:
- What the symptom was
- Root cause
- Fix applied

This applies to both code bugs and infrastructure issues (RLS, Realtime, migrations, etc).

## BMAD Workflow Phases

Work progresses through four sequential phases, each producing artifacts in `_bmad-output/`:

| Phase | Goal | Output Location |
|-------|------|-----------------|
| 1. Analysis | Research, brainstorm, product brief | `_bmad-output/planning-artifacts/` |
| 2. Planning | PRD, UX design | `_bmad-output/planning-artifacts/` |
| 3. Solutioning | Architecture, epics & stories | `_bmad-output/planning-artifacts/` |
| 4. Implementation | Sprint plan, story execution, code review | `_bmad-output/implementation-artifacts/` |

## Available Skills

Invoke skills with the Skill tool (e.g., `/bmad-quick-spec`). Key skills:

- `bmad-quick-spec` / `bmad-quick-dev` ‚Äî Fast spec and implementation for small changes
- `bmad-create-prd` / `bmad-create-architecture` / `bmad-create-epics-and-stories` ‚Äî Full project planning
- `bmad-dev-story` ‚Äî Implement a story from a spec file in `_bmad-output/`
- `bmad-code-review` ‚Äî Multi-layer adversarial review (Blind Hunter, Edge Case Hunter, Acceptance Auditor)
- `bmad-sprint-planning` / `bmad-sprint-status` ‚Äî Sprint tracking
- `bmad-document-project` ‚Äî Document a brownfield codebase for AI context

Maintained by the tech-writer agent (see `_bmad/_memory/tech-writer-sidecar/documentation-standards.md`):
- Use CommonMark markdown
- Mermaid diagrams use v10+ syntax
- Do not include time estimates in documentation
