# Badminton V2 — Project Instructions

Inherits all rules from global `~/.claude/CLAUDE.md`.

## Project Context

This project uses the **BMAD framework (v6.2.0)** — an AI-driven project orchestration system. It provides structured workflows for taking a project idea through analysis, planning, solutioning, and implementation phases using role-based agents.

## Task Management

- Plans go in `tasks/todo.md`
- Lessons learned go in `tasks/lessons.md`

## BMAD Workflow Phases

Work progresses through four sequential phases, each producing artifacts in `_bmad-output/`:

| Phase | Goal | Output Location |
|-------|------|-----------------|
| 1. Analysis | Research, brainstorm, product brief | `_bmad-output/planning-artifacts/` |
| 2. Planning | PRD, UX design | `_bmad-output/planning-artifacts/` |
| 3. Solutioning | Architecture, epics & stories | `_bmad-output/planning-artifacts/` |
| 4. Implementation | Sprint plan, story execution, code review | `_bmad-output/implementation-artifacts/` |

## Available Skills

Invoke skills with the Skill tool (e.g., `/bmad-quick-spec`). Key skills:

- `bmad-quick-spec` / `bmad-quick-dev` — Fast spec and implementation for small changes
- `bmad-create-prd` / `bmad-create-architecture` / `bmad-create-epics-and-stories` — Full project planning
- `bmad-dev-story` — Implement a story from a spec file in `_bmad-output/`
- `bmad-code-review` — Multi-layer adversarial review (Blind Hunter, Edge Case Hunter, Acceptance Auditor)
- `bmad-sprint-planning` / `bmad-sprint-status` — Sprint tracking
- `bmad-document-project` — Document a brownfield codebase for AI context

## Key Agents

Agents are role-based personas used inside workflows:

- **analyst (Mary)** — Research, competitive analysis, requirements
- **architect (Winston)** — System design, ADRs, scalability
- **dev (Amelia)** — Story implementation, TDD
- **pm (John)** — PRD creation, stakeholder alignment
- **qa (Quinn)** — Test automation, coverage
- **sm (Bob)** — Scrum master, story preparation
- **ux-designer (Sally)** — User research, interaction design
- **tech-writer (Paige)** — Documentation (CommonMark, Mermaid v10+)

## Documentation Standards

Maintained by the tech-writer agent (see `_bmad/_memory/tech-writer-sidecar/documentation-standards.md`):
- Use CommonMark markdown
- Mermaid diagrams use v10+ syntax
- Do not include time estimates in documentation

## Framework Layout

- `_bmad/` — BMAD framework modules (core, bmm, bmb, cis, tea); do not edit directly
- `_bmad-output/` — All generated artifacts from workflow execution
- `_bmad/_config/skill-manifest.csv` — Index of all 50+ available skills
- `docs/` — Project documentation (human-facing, generated by tech-writer)
